Aanbevelingen - Fase 1: Gebruikers, licenties en basisbeveiliging

Microsoft best practices voor gebruikersbeheer, licenties en basisbeveiliging:

⚠️ MFA-afdwinging (kritiek): 3 gebruiker(s) hebben geen MFA geregistreerd.
Microsoft best practice: Implementeer MFA voor alle gebruikers via Conditional Access-beleid. Gebruik waar mogelijk wachtwoordloze authenticatie (Windows Hello, FIDO2, Authenticator-app).
👥 Gastgebruikers (1):
Microsoft best practice: Implementeer een gasttoegangsbeleid: (1) beoordeel gastgebruikers elke 3-6 maanden, (2) gebruik toegangsbeoordelingen voor automatische verificatie, (3) beperk gastrechten via Azure AD External Identities-instellingen, (4) monitor gastactiviteit via auditlogs.
⚠️ CA↔MFA-mismatch (kritiek): 9 gebruiker(s) vallen binnen Conditional Access-beleid dat MFA vereist, maar hebben geen geregistreerde MFA-methoden.
Microsoft best practice: Corrigeer deze gebruikers: forceer MFA via Conditional Access (GrantControls: Require multifactor authentication) in plaats van per-user uitzonderingen. Gebruik phishing-resistente methoden (FIDO2) voor kritieke accounts en monitor aanmeldlogs.

Algemene best practices - Fase 1

  • Identiteitsbescherming: Schakel Azure AD Identity Protection in voor risicogebaseerde Conditional Access
  • Wachtwoordbeleid: Gebruik Azure AD Password Protection om zwakke wachtwoorden te blokkeren
  • Beheeraccounts: Gebruik aparte beheeraccounts (gescheiden van reguliere accounts)
  • Monitoring: Configureer waarschuwingen voor beheerrolwijzigingen en risicovolle aanmeldingen
  • Lifecyclemanagement: Implementeer in-, door- en uitstroomprocessen voor gebruikersbeheer
  • Licentiebeheer: Gebruik waar mogelijk automatische licentietoewijzing via groepen
